BootStrap学习笔记五

1. JavaScript插件

在BootStrap3中javascript插件依赖于jQuery,导入bootstrap.min.js即可

jQuery 插件为 Bootstrap 的组件赋予了“生命”。可以简单地一次性引入所有插件，或者逐个引入到你的页面中。

2. data属性

你可以仅仅通过 data 属性 API 就能使用所有的 Bootstrap 插件，无需写一行 JavaScript 代码。这是 Bootstrap 中的一等 API，也应该是你的首选方式！

在某些情况下可能需要将此功能关闭。BootStrap还提供了关闭 data 属性 API 的方法，即解除以 data-api 为命名空间并绑定在文档上的事件。就像下面这样：

$(document).off('.data-api')

如果是针对某个特定的插件，只需在 data-api 前面添加那个插件的名称作为命名空间，如下：

$(document).off('.alert.data-api')

详解可看下面的链接，

https://v3.bootcss.com/javascript/#js-overview

3. 模态框 modal.js

即可以弹出一个对话框

不支持同时打开多个模态框

千万不要在一个模态框上重叠另一个模态框。要想同时支持多个模态框，需要自己写额外的代码来实现。

所有JavaScript组件都有三个知识点：属性，方法，事件

模态框的 HTML 代码放置的位置

务必将模态框的 HTML 代码放在文档的最高层级内（也就是说，尽量作为 body 标签的直接子元素），以避免其他组件影响模态框的展现和/或功能。

创建一个模态框

静态实例

以下模态框包含了模态框的头、体和一组放置于底部的按钮。

<!-- 创建一个模态框 modal
		1. 在一张页面中不支持同时打开多个模态框 千万不要在一个模态框上重叠另一个模态框。
		2. 务必将模态框的HTML代码放在文档的最高层级内（即作为body标签的直接子元素）
	-->
	<div class="modal fade"><!-- fade代表呈现模态框时呈渐变 -->
		<!-- modal-dialog -->
		<div class="modal-dialog">
			<!-- modal-content -->
			<div class="modal-content">

				<!-- modal-header:模态框头 -->
				<div class="modal-header">
					<!-- 创建一个关闭按钮用于关闭模态框 -->
					<button type="button" class="close" data-dismiss="modal">
						<span>&times;</span>
					</button>

					<!-- 写标题:modal-title -->
					<h4 class="modal-title">模态框标题</h4>
				</div>

				<!-- modal-body:模态框的体 -->
				<div class="modal-body">
					<p>这是一个模态框的内容</p>
				</div>

				<!-- modal-footer:模态框底部 主要用于定制按钮 -->
				<div class="modal-footer">
					<button type="button" class="btn btn-danger" data-dismiss="modal">关闭</button>
					<button type="button" class="btn btn-primary">确认</button>
				</div>

			</div>
		</div>
	</div>

[注意]：上面的代码在模态框中默认是不显示的

动态实例

由于只创建模态框时无法显示的，所以可以创建一个按钮用于启动模态框，在上面代码的基础上加入以下代码：

<!-- 创建一个按钮用于点击启动模态框
	只是单纯的创建一个模态框默认是不显示的
-->
<button type="button" class="btn btn-primary" data-toggle="modal" data-target="#mymodal">
	点击启动模态框
</button>

可选尺寸

模态框提供了两个可选尺寸，通过为 .modal-dialog 增加一个样式调整类实现。

.modal-lg：大模态框

.modal-sm：小模态框

<!-- 在modal-dialog 后面加入这两个类即可 -->
<div class="modal-dialog modal-lg">
   ...
</div>
### 禁止动画效果

如果你不需要模态框弹出时的动画效果（淡入淡出效果），删掉 .fade 类即可。

<div class="modal" tabindex="-1" role="dialog" aria-labelledby="...">
  ...
</div>

栅格系统

模态框中也可以使用栅格系统

用法

通过 data 属性或 JavaScript 调用模态框插件，可以根据需要动态展示隐藏的内容。模态框弹出时还会为 <body> 元素添加 .modal-open 类，从而覆盖页面默认的滚动行为，并且还会自动生成一个 .modal-backdrop 元素用于提供一个可点击的区域，点击此区域就即可关闭模态框。

  1. 通过data属性

不需写 JavaScript 代码也可激活模态框。通过在一个起控制器作用的元素（例如：按钮）上添加 data-toggle="modal" 属性，或者 data-target="#foo" 属性，再或者 href="#foo" 属性，用于指向被控制的模态框。

<button type="button" data-toggle="modal" data-target="#myModal">Launch modal</button>
  1. 通过JavaScript调用

只需一行 JavaScript 代码，即可通过元素的 id 调用模态框：

参数

可以将选项通过 data 属性或 JavaScript 代码传递。对于 data 属性，需要将参数名称放到 data- 之后，例如 data-backdrop=""。

名称 类型 默认值 描述
backdrop boolean 或 字符串 'static' true 包含模态背景元素。或者，为点击时不关闭模式的背景指定静态。即点击模态框以外的区域不会关闭模态框
keyboard boolean true 键盘上的 esc 键被按下时关闭模态框。（不知为何没有反应）
focus boolean true 初始化时将重点放在模态上。
show boolean true 模态框初始化之后就立即显示出来。

方法

异步方法和转换

所有 API 方法都是异步的并开始转换。一旦转换开始但在结束之前，它们就立即返回给调用者。另外，对转换组件的方法调用将被忽略。

语法如下：

.modal(options)

1 将页面上的某块内容作为模态框激活。接受可选参数object

<script>
	$(function () {
		//页面加载之后立即展示模态框
		$('#mymodal').modal({
			show: true, //展示模态框
			backdrop: 'static' //点击旁边区域不会关闭模态框
		})
	})
</script>

2 .modal('toggle')：手动打开或关闭模态框。在模态框显示或隐藏之前返回到主调函数中（也就是，在触发 shown.bs.modal 或 hidden.bs.modal 事件之前）。

$('#myModal').modal('toggle')

3 .modal('show')：手动打开模态框。在模态框显示之前返回到主调函数中 （也就是，在触发 shown.bs.modal 事件之前）。

$('#myModal').modal('show')

  1. .modal('hide')：手动隐藏模态框。在模态框隐藏之前返回到主调函数中 （也就是，在触发 hidden.bs.modal 事件之前）。

$('#myModal').modal('hide')

  1. .modal('handleUpdate')：如果模态框的高度在打开时发生变化（例如出现滚动条），则手动重新调整模态的位置。

$('#myModal').modal('handleUpdate')

事件

Bootstrap 的模态框类提供了一些事件用于监听并执行你自己的代码。

所有的模态事件都是在模态本身（即在<div class="modal">）处被触发的。

事件类型 描述
show.bs.modal show 方法调用之后立即触发该事件。如果是通过点击某个作为触发器的元素，则此元素可以通过事件的 relatedTarget 属性进行访问。
shown.bs.modal 此事件在模态框已经显示出来（并且同时在 CSS 过渡效果完成）之后被触发。如果是通过点击某个作为触发器的元素，则此元素可以通过事件的 relatedTarget 属性进行访问。
hide.bs.modal hide 方法调用之后立即触发该事件。
hidden.bs.modal 此事件在模态框被隐藏（并且同时在 CSS 过渡效果完成）之后被触发。 
$('#myModal').on('hidden.bs.modal', function (e) {
  // do something...
})

4. 标签页

<div>

  <!-- 创建一个标签页 -->
  <ul class="nav nav-tabs">
    <!-- 超链接为下方tab-pane的id值 -->
    <li class="active"><a href="#home" data-toggle="tab">Home</a></li>
    <li><a href="#profile"  data-toggle="tab">Profile</a></li>
    <li><a href="#messages" data-toggle="tab">Messages</a></li>
    <li><a href="#settings" data-toggle="tab">Settings</a></li>
  </ul>

  <!-- 创建每个标签页相对应的内容容器：tab-content -->
  <div class="tab-content">
    <div class="tab-pane active" id="home">...</div>
    <div class="tab-pane" id="profile">...</div>
    <div class="tab-pane" id="messages">...</div>
    <div class="tab-pane" id="settings">...</div>
  </div>

</div>

你也可以使用nav-pill，<a>标签中要修改为data-toggle="pill"

渐入渐出效果

与模态框类似，只要加入.fade类到每个tab-pane下即可，对于第一个tab-pane则需要加入.in类使其有效

<div class="tab-content">
  <div class="tab-pane fade in active" id="home">...</div>
  <div class="tab-pane fade" id="profile">...</div>
  <div class="tab-pane fade" id="messages">...</div>
  <div class="tab-pane fade" id="settings">...</div>
</div>

方法

.tab('show')：显示目标标签页

$('#myBtn').click(function(){
    	// homes必须是<a>标签中的id
		$('#homes').tab('show');
	})
})

事件：

show.bs.tab
shown.bs.tab
hide.bs.tab
hidden.bs.tab

这四个事件与模态框事件类似
